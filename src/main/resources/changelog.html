Changelog

v1.0.5 - Remote File Browser & Rollback

  • 🔄 Deployment Rollback: One-click rollback to previous version for mappings with backup enabled. Rollback button appears in history for deploy records with backups, with full confirmation dialog and progress tracking. All rollback operations are logged in history with full reports
  • 📂 Remote File Browser: Browse remote server file systems in a sidebar tree (DeployX Remote tool window, right side dock). All operations are non-modal — browse remote files while editing local projects
  • 👀 View: Lazy-loaded directory tree with file size, modification time and icons; switch servers, path navigation with per-server history (up to 20 entries, jump / delete individual / clear all), create/delete directories, refresh
  • 📝 Edit: Double-click a remote text file to open it in the IDE main editor with full syntax highlighting; Ctrl+S automatically writes changes back to the remote server; files over 5MB are rejected with a download suggestion
  • ⬆️ Upload: Drag local files/directories from the project view or OS file manager onto the browser tree to upload to the target remote directory
  • ⬇️ Download: Right-click to download files/directories with a progress dialog; or drag remote entries from the tree to the project view to download directly
  • 🎯 Auto-map & Remember: First open auto-locates the remote directory by matching the current project path against configured mappings; each server's last-opened directory is remembered across sessions
  • Persistent SFTP channel with auto-reconnect and automatic I/O retry. Accessible via right-click menu, Alt+Shift+Z quick menu, and the folder button in the DeployX operation tab. Full bilingual (English / 简体中文) support
  • 📝 Script Editor Reuse: Migrated all script/command editing UIs from Swing native components (JBTextArea + hand-written LineNumberGutter) to the IDEA platform EditorTextField, with syntax highlighting, line numbers, code folding, current-line highlight and automatic theme adaptation. Fixed Enter key triggering the OK button in dialogs, multi-line paste collapsing to one line, and Tab key being swallowed by focus traversal
  • 🗑️ Removed LineNumberGutter: Deleted the hand-written LineNumberGutter.kt — the platform editor's built-in gutter fully replaces it
  • 🔐 SSH Key Permission Fix: Fixed key authentication failing in file sync/deploy with Permissions 0644 for 'xxx' are too open when terminal works fine. Root cause: the terminal uses JSch (doesn't check file permissions), while rsync uses system ssh (OpenSSH enforces strict key permissions). Fix: auto-detects overly-permissive key files, creates a temp copy with correct 0600 permissions, uses it for transfer, then cleans up. Users don't need to manually fix key file permissions anymore
  • 🐛 Update Report Mis-detects GLIBC/Library Errors: Fixed dynamic-library error lines printed by the remote shell or toolchain (mise/asdf etc.) due to missing GLIBC versions (e.g. /lib64/libc.so.6: version 'GLIBC_2.18' not found (required by ...)) being misclassified as transferred files by isFileTransferLine, inflating the report's "total files" count and listing the error message under "actually updated files". Added substring-based diagnostic filtering (not found, required by, glibc, libc.so, shared object, etc.)
  • 🐛 Meaningless Backup File Names: Fixed backup files being uniformly named backup_{timestamp}.tar.gz with no indication of what was backed up. Now named with the backed-up file/dir name (last segment of the first relative path, truncated) + _bak + date, e.g. updates_bak_20260712_025456.tar.gz; multiple files use xxx_nfiles_bak_{timestamp}.tar.gz
  • 🐛 Missing _bak_ in Backup Names: Fixed doBackup naming tar.gz backups of directories/non-compressed files without the _bak_ identifier ({name}_{timestamp}.tar.gz), inconsistent with other backup branches. Also fixed double-extension archives (e.g. foo.tar.gz) being split incorrectly into foo.tar_bak_...gz
  • 🐛 Backup Path Returns Directory Instead of File: Fixed doBackup returning the backup directory path (instead of the specific backup file path) on success, causing the update report's "backup location" to show only the directory and rollback's test -f check to fail. Now consistently returns the specific backup file path
  • 🐛 Clear History Button Icon Misleading: Fixed the History tab "Clear" button using a history icon (AllIcons.Vcs.History, a clock) whose semantics contradict "clear all history" and is easily mistaken for "view history". Changed to the trash icon (AllIcons.Actions.GC); also replaced the top "Clear Log" button's custom gray icon with the platform AllIcons.Actions.GC so its color adapts to the theme and matches the rest of the toolbar

v1.0.4 - SSH Key Export & New Features

  • 🔑 SSH Key Export/Import: When exporting encrypted config, if any servers use SSH key authentication, users are prompted whether to include encrypted key files in the export. On import, key files are automatically decrypted and saved to the local keys directory (~/.deploy-x/keys/), with server key paths updated to point to the local files. If the user chooses not to export key files, only server config is exported (key paths preserved but content not included)
  • 🐛 Config Import Lost Server Passwords: Fixed an issue where imported servers had an empty password in memory after config import, causing SSH authentication to fail. The root cause was that ConfigExporter.importConfig passed server.copy(password = "") to ServerManager after saving the password to PasswordSafe / .passwords.dat, clearing the password from the in-memory ServerConfig. Now the full ServerConfig with password is passed through directly, and saveServers() continues to sanitize the password when writing servers.json, matching existing behavior
  • 📁 New Folder Button in Remote Path Chooser: Added a "New Directory" button in the remote path selection dialog (used in add/edit mapping and sidebar operations). Users can create directories directly on the remote server without switching to the terminal. After successful creation, the directory list is automatically refreshed and the new directory is selected.
  • 📏 Adaptive Server Dropdown Width: The target server dropdown in the add/edit mapping dialog now automatically adapts to the width of the longest server name, ensuring the server ID and name are fully visible without truncation.
  • 🔄 Bidirectional Sync (Pull from Server): New bidirectional incremental sync between local and remote files with auto-merge of differences. Right-click Pull from Server (or the Alt+Shift+Z quick menu) downloads only the files that differ on the remote to local. Uses rsync when available (transfers only changed files by size/mtime), with automatic SFTP fallback for recursive directory download
  • 🐛 rsync Pull Direction Reversed: Fixed Pull from Server in rsync mode generating a --files-from command that used the local dir as source and the remote dir as target, actually uploading instead of pulling. Command builders now swap source/target based on SyncDirection.PULL
  • 🐛 Unclear Pull Log: Fixed the download log reusing the "Preview group" header. The download group now uses a dedicated "Pull from Server group" header and prints direction, remote source dir and local target dir explicitly
  • 🐛 Alt+Shift+Z Missing Pull from Server: Added DeployX.PullFromServer to the Alt+Shift+Z quick menu (after Sync to Server)
  • 🐛 rsync Pull Directory Transfers 0 Files: Fixed pulling a directory/mapping-root (empty relative path) producing an empty --files-from entry so rsync transferred 0 files. Empty entries now do a full recursive directory pull; non-empty entries still use --files-from; both can be mixed and merged. SFTP fallback handles empty relative paths the same way
  • 🐛 History Tab Blank: Fixed the History tab showing blank after the tool window's first build (when HistoryManager wasn't ready). Now reloads latest records on each switch to the History tab, and shows a "No history yet" placeholder when empty
  • 🐛 Pull from Server not Recording History: Fixed "Pull from Server" (the download path of bidirectional sync) leaving no history entry after execution. Root cause: downloadBatch / processDownloadGroup only transferred files and never called HistoryManager.addRecord, and HistoryRecord.OperationType had no pull type. Added a PULL type and writes history on pull success/failure (only real pulls, not dry-run preview); the history summary shows "remote → local" for PULL; "redeploy" for a PULL record now re-pulls (remote → local); UpdateReportFormatter renders the new PULL operation type
  • 🐛 Sync to Server / Push to Server not Recording History: Fixed right-click "Sync to Server" / "Push to Server" leaving no history entry. Root cause: those paths go through uploadBatch → processUploadGroup, but history writing only existed in deployBatch's processDeployGroup. Added saveUploadGroupHistory (type = UPLOAD) inside processUploadGroup, writing on non-dry-run success/failure; preview (dry-run) explicitly does not write history
  • 🐛 History List Single-click Mis-jumped to Log Tab: Fixed single-clicking a history entry immediately jumping to the Log tab, preventing the entry from being selected. Root cause: addListSelectionListener triggered showHistoryDetail() on selection, whose appendLog switches to the Log tab. Changed to a MouseListener that only views details and jumps to the Log tab on double-click; single-click only selects, without affecting later "redeploy / copy report / fill config" operations
  • 🐛 Server Dropdown Too Narrow: Fixed the target server dropdown in the add/edit mapping dialog being clamped to the field width so server id/name were truncated. Now uses a wide popup (ComboBoxWithWidePopup) that expands to fit the longest entry instead of being limited to the input box width
  • 📖 "What's New" Settings Tab: Added a "What's New" tab to the DeployX settings page (right after "Script Library") showing the version changelog. It displays only the changelog for the current UI language (English in English mode, 简体中文 in Chinese mode), reusing a single changelog source shared with the plugin marketplace
  • 🐛 Settings Page Over-wide / Other Tabs Distorted: Fixed the settings page showing an extremely long horizontal scrollbar that also stretched and distorted the tables on other tabs (e.g. Script Library). Root cause: the "What's New" tab's HTML pane pushed an oversized preferred width into the tabbed pane. The tab now caps its preferred size so content scrolls only inside the tab and never widens the whole dialog

v1.0.3 - Transfer Enhancements & Multi-Server Deploy

  • 📋 History Report Copy/Export: Added "Copy Report" and "Export Report" buttons to the History tab - copy/export the Markdown report from any history entry (not just the latest one)
  • 🔍 SFTP Dry-Run Support: SFTP fallback mode now supports dry-run preview (listing files that would be uploaded without actual transfer). Preview sync works even without rsync installed
  • 🔄 Transfer Auto-Retry: Transfers now automatically retry up to 3 times (2s interval) on network/connection failures. Authentication failures and missing-path errors are not retried
  • 🔔 System Notifications: After batch upload/deploy completes, an OS-level notification is shown when the IDE window is not focused (Linux via libnotify, macOS via Notification Center, Windows 10/11 via PowerShell WinRT Toast), plus an IDE balloon is always shown. Configurable in settings. On Windows, replaced the platform SystemNotifications path (which uses the deprecated java.awt.TrayIcon balloon that silently fails when SystemTray.isSupported() returns false or AWTException is swallowed) with a reliable WinRT Toast notifier
  • ⚙️ General Settings Tab: Added a new "General" tab at the front of the DeployX settings page, consolidating language and the "system notification on transfer complete" toggle (previously buried in the rsync tab) into one place, ready for more general preferences to be added later
  • 🏷️ Server Groups & Tags: Servers now support a group field and comma-separated tags. ServerManager exposes getGroups()/getAllTags() for filtering
  • 📦 Config Import/Export (Encrypted): Export all plugin config (servers with passwords, mappings, scripts) to an AES-256-GCM encrypted JSON file. Import on another machine with password decryption; ID conflicts can be overwritten or imported as new
  • ⚡ Multi-Server Parallel Deploy: Server selection dialog now supports multi-select. When deploying to multiple servers, groups execute in parallel (thread pool, max 4 concurrent). HistoryManager is thread-safe
  • 🐛 Server List Empty Fix: Fixed an issue where users saw an empty server management page. The new group/tags fields were null in legacy servers.json, causing an NPE in server.copy() that silently wiped the server list. A ServerConfigDeserializer now backfills missing/null fields

v1.0.2 - Script Library & Windows rsync Enhancements

  • 📚 Script Export: Export now adapts to selection - exports all scripts when nothing is selected, and only the selected scripts (as a JSON array) when multiple are selected
  • 🗑️ Script Multi-Delete: Deleting now removes all selected scripts at once (previously only the first selected was deleted)
  • 📥 Script Import - Conflict Handling: On import, if a script id already exists, the user is prompted to either overwrite the duplicate or import all as new scripts
  • 🔐 SSH_ASKPASS Fallback: When rsync is present but sshpass is missing (common on Windows), password auth automatically falls back to the SSH_ASKPASS mechanism - no SFTP downgrade needed
  • ⬇️ rsync One-Click Download & Install (Windows): When rsync is not detected on Windows, a "Download & Install rsync" button appears in settings; it auto-downloads (GitHub releases, with China mirror fallback), extracts to ~/.deployx/rsync-win/, and configures the path automatically. On failure, manual download links are provided
  • 💡 First-Run rsync Install Prompt (Windows): On the first transfer with rsync missing, a dialog asks whether to auto-install. "Yes" downloads and installs; "No" remembers the choice and falls back to SFTP silently (never asks again). RSYNC_ONLY mode also degrades to SFTP after the user declines
  • 🪟 Windows rsync Optimization: Fixed rsync failing on Windows Cygwin builds:
    • Quote the ssh command passed to -e so it is not split incorrectly
    • Convert Windows drive paths (e.g. C:\Users\...) to Cygwin style (/cygdrive/c/Users/...) to avoid the drive letter being mistaken for a remote host
    • Use --files-from=FILE with a temporary file (auto-deleted after use) instead of stdin piping
  • 🐛 Windows rsync Password Auth Fix: Fixed exit code 12 / 0 bytes received when syncing with password authentication on Windows without sshpass (three root causes, all verified and fixed):
    • askpass script format: Minimal Cygwin rsync packages lack sh.exe; cygwin ssh invokes askpass via posix_spawnp which does not parse shebangs, so .sh scripts fail with No such file or directory. Switched to a .cmd askpass (@type "path") executed via cmd.exe /c
    • wrong ssh picked: rsync -e "ssh ..." resolved to the Windows native C:\Windows\System32\OpenSSH\ssh.exe instead of the Cygwin ssh, breaking SSH_ASKPASS compatibility. Now explicitly uses the ssh.exe next to rsync.exe
    • missing recursion: -a does not imply -r under --files-from, so selecting a directory only created empty subdirectories without transferring the files inside. Now always adds an explicit -r
  • 🐛 Directory Sync Fix (all platforms): Fixed selecting a directory and syncing only created empty subdirectories without transferring the files inside (same -a/-r root cause as above, affects Linux/macOS too)
  • 🐛 Update Report Fixes:
    • Fixed SSH diagnostic lines (e.g. Warning: Permanently added ...) being misidentified as transferred files in the report
    • Fixed rsync directory creation lines (ending with /) being misidentified as transferred files - report now lists only actual files, matching rsync's Number of regular files transferred
    • Fixed "Actually updated files" showing selected directories or unchanged files when 0 files were transferred (incremental skip) - now shows "No files changed" instead
  • ⚡ Multi-Server Batch Deploy: Support selecting multiple servers at the same time, files will be uploaded to all selected servers in parallel, greatly improving the efficiency of multi-environment deployment; the server selection dialog supports holding Ctrl/Shift for multiple selection, and the terminal also supports opening multiple server connections at the same time
  • 🎨 SSH Clean-up: Added -o UserKnownHostsFile=/dev/null to suppress the Could not create directory '/home/<user>/.ssh' warning from Cygwin ssh on minimal installs
  • 🎨 Alt+Shift+Z Menu Fix: Fixed the quick menu popping up in the wrong IDEA window when multiple project windows are open - now uses guessBestPopupLocation(dataContext) for correct positioning
  • 🎨 Changelog Unification: Synced build.gradle.kts and plugin.xml changelog content - plugin marketplace now shows the full bilingual version

v1.0.4 - SSH 密钥导出与新功能增强

  • 🔑 SSH 密钥导出/导入：导出加密配置时，若存在使用密钥认证的服务器，会提示用户是否将密钥文件内容一并加密导出。导入到另一台机器时自动解密并保存到本地密钥目录（~/.deploy-x/keys/），服务器配置的密钥路径自动指向本地文件，无需手动复制密钥。若不选择导出密钥文件，仍按原有方式仅导出服务器配置（密钥路径保留但内容不导出）
  • 🐛 配置导入后服务器丢失密码修复：修复导入加密配置后，服务器密码在内存中丢失导致 SSH 连接失败的问题。根因：ConfigExporter.importConfig 在保存密码到 PasswordSafe / .passwords.dat 后，传给 ServerManager 的是 server.copy(password = "") 的副本，导致内存中的 ServerConfig.password 为空，SSH 密码认证失败。现已改为直接传入带密码的 ServerConfig，saveServers() 仍会在写入 servers.json 时清空密码字段，行为与现有逻辑完全一致
  • 📁 远程路径选择对话框新增新建目录按钮：在添加/编辑映射、侧边栏操作的远程路径选择对话框中，新增新建目录功能，支持直接在远程服务器上创建目录，无需切换到终端手动操作，创建成功后自动刷新目录列表并选中新目录。
  • 📏 服务器下拉列表宽度自适应：在添加/编辑映射对话框中，目标服务器下拉列表自动适配最长服务器名称的宽度，确保完整显示服务器ID和名称，避免内容截断。
  • 🔄 双向同步（从服务器拉取）：新增本地与远程文件的双向增量同步，自动合并差异。右键「从服务器拉取」或使用 Alt+Shift+Z 快捷菜单，可将远程与本地不同的文件拉取到本地；rsync 模式按大小/修改时间只传输差异文件，SFTP 降级时递归下载整个远程目录
  • 🐛 rsync 拉取方向反转：修复「从服务器拉取」使用 rsync 模式时，生成的 --files-from 命令把本地当源、远程当目标，实际变成上传。命令构建现在按 SyncDirection.PULL 动态交换源/目标
  • 🐛 拉取日志不清晰：修复下载日志误用「预览分组」标题。下载分组现使用独立的「从服务器拉取分组」标题，并明确打印方向、远程源目录与本地目标目录
  • 🐛 Alt+Shift+Z 菜单缺少「从服务器拉取」：已将 DeployX.PullFromServer 加入 Alt+Shift+Z 快捷菜单（位于「同步到服务器」之后）
  • 🐛 rsync 拉取目录/映射根时文件数为 0：修复选中目录/映射根（相对路径为空）时 --files-from 列表为空条目、rsync 实际传输 0 文件的问题。空项现走整目录递归拉取，非空项仍走 --files-from，二者可混合并合并结果；SFTP 降级模式同样处理
  • 🐛 历史 Tab 展示空白：修复侧边面板「历史」Tab 初次构建时因 HistoryManager 未就绪而加载空列表、之后一直空白的问题。现切换至历史 Tab 时自动重新加载最新记录，列表为空时显示「暂无历史记录」占位提示
  • 🐛 「从服务器拉取」不记录历史：修复「从服务器拉取」（双向同步的下载路径）执行后历史面板没有任何记录的问题。根因：downloadBatch / processDownloadGroup 只负责传输，从不调用 HistoryManager.addRecord，且 HistoryRecord.OperationType 没有拉取类型。现已新增 PULL 类型，并在拉取成功/失败后写入历史（仅实际拉取写，dry-run 预览不写），历史列表摘要对 PULL 显示「远程 → 本地」方向；历史「重新部署」对 PULL 记录改为执行「重新拉取」（远程 → 本地），更新报告 operationType 新增 PULL 渲染
  • 🐛 「同步到服务器 / 推送到服务器」不记录历史：修复右键「同步到服务器」「推送到服务器」执行后历史面板没有记录的问题。根因：这两条路径走 uploadBatch → processUploadGroup，而历史写入只存在于 deployBatch 的 processDeployGroup 中。现已在 processUploadGroup 内新增 saveUploadGroupHistory（type = UPLOAD），仅在非 dry-run 的实际同步/推送成功或失败后写入历史；预览（dry-run）明确不写历史
  • 🐛 历史列表单击误跳转日志页：修复在「历史」Tab 中单击某条记录会立即跳转到「日志」页签、无法单独选中的问题。根因：addListSelectionListener 在单击选中时就触发 showHistoryDetail()，其内部 appendLog 会切到日志 Tab。现改为 MouseListener 双击才查看详情并跳转「日志」页签；单击仅做选中，不影响后续「重新部署 / 复制报告 / 填充配置」等操作
  • 🐛 映射服务器下拉过窄：修复添加/编辑映射时「目标服务器」下拉框被输入框宽度钳制、服务器 id/名称显示不全的问题。现改用可超出输入框宽度的宽弹层（ComboBoxWithWidePopup），自动适配最长一项，不再被表单列宽截断
  • 📖 「版本更新说明」设置 Tab：在 DeployX 设置页「脚本库」之后新增「版本更新说明」Tab，展示版本更新日志。按当前界面语言只显示对应语言内容（英文模式显示英文、中文模式显示中文），更新内容与插件市场共用同一单一来源，无需多处维护。
  • 🐛 设置页横向滚动条过长/其他 Tab 表格变形：修复设置页出现超长横向滚动条、并导致「脚本库」等其他 Tab 表格被拉变形的问题。根因：「版本更新说明」Tab 的 HTML 面板推出了超大首选宽度。现已对 Tab 封顶首选尺寸，内容仅在 Tab 内部滚动，不再撑大整个对话框、影响其他 Tab 布局。

v1.0.2 - 脚本库增强与 Windows rsync 优化

  • 📚 脚本导出优化：导出逻辑适配选中状态 - 未选中任何脚本时导出全部脚本，选中多个脚本时仅导出选中的脚本（JSON 数组形式）
  • 🗑️ 脚本批量删除：删除操作现在可一次性删除所有选中的脚本（此前只会删除选中的第一个）
  • 📥 脚本导入冲突处理：导入时若脚本 id 已存在，提示用户选择「覆盖重复项」或「全部作为新脚本新增」
  • 🔐 SSH_ASKPASS 密码认证回退：已安装 rsync 但缺少 sshpass 时（Windows 常见），密码认证自动回退使用 SSH_ASKPASS 机制，无需降级 SFTP
  • ⬇️ rsync 一键下载安装（Windows）：设置面板检测到未安装 rsync 时显示「一键下载安装」按钮，自动下载（GitHub releases，失败切换国内镜像）、解压到 ~/.deploy-x/rsync-win/、自动配置路径；下载失败提供手动下载链接
  • 💡 首次传输安装提示（Windows）：首次传输时未检测到 rsync 则弹窗询问是否自动安装。选「是」下载安装；选「否」记住选择并静默降级 SFTP（不再提示）。RSYNC_ONLY 模式下用户拒绝后也降级 SFTP
  • 🪟 Windows rsync 同步优化：修复 Windows Cygwin 版 rsync 同步失败的问题：
    • -e 后的 ssh 命令整体加引号，避免被错误拆分
    • Windows 盘符路径（如 C:\Users\...）转换为 Cygwin 风格（/cygdrive/c/Users/...），避免盘符被误识别为远程主机
    • --files-from 改用临时文件（使用完自动删除）替代 stdin 传入
  • 🐛 Windows rsync 密码认证修复：修复 Windows 无 sshpass 时密码认证同步报 exit code 12 / 0 bytes received 的问题（三个根因，均已实测验证修复）：
    • askpass 脚本格式：精简 Cygwin rsync 包不含 sh.exe，cygwin ssh 通过 posix_spawnp 调用 askpass（不解析 shebang），.sh 脚本因找不到 /bin/sh 报 No such file or directory。改用 .cmd 脚本（@type "路径"），由 cmd.exe /c 执行
    • ssh 来源错误：rsync -e "ssh ..." 用裸 ssh 时会找到 Windows 原生 C:\Windows\System32\OpenSSH\ssh.exe 而非 cygwin ssh，导致 SSH_ASKPASS 机制不兼容。改为显式使用 rsync.exe 同目录下的 ssh.exe
    • 缺少递归：-a 在 --files-from 模式下不会展开为 -r，选中目录同步时只创建空子目录、不传文件。现已显式补加 -r
  • 🐛 目录同步修复（全平台）：修复选中目录同步时只创建空子目录、不传输目录内文件的问题（同上 -a/-r 根因，Linux/macOS 同样受影响）
  • 🐛 更新报告修复：
    • 修复 SSH 诊断行（如 Warning: Permanently added ...）被误判为传输文件、拼成远程路径写入报告的问题
    • 修复 rsync 目录创建行（以 / 结尾）被误判为传输文件的问题 - 报告现在只列出实际传输的文件，与 rsync 统计 Number of regular files transferred 一致
    • 修复增量同步跳过所有文件（0 传输）时报告"实际更新的文件"回退到选中路径（含目录）的问题 - 现在显示"无文件变更"提示
  • 🎨 SSH 优化：新增 -o UserKnownHostsFile=/dev/null，消除精简 Cygwin 环境下 Could not create directory '/home/<user>/.ssh' 警告
  • 🎨 Alt+Shift+Z 菜单修复：修复多个 IDEA 项目窗口时快捷菜单弹到错误窗口的问题 - 改用 guessBestPopupLocation(dataContext) 正确定位
  • 🎨 Changelog 统一：统一 build.gradle.kts 与 plugin.xml 的 changelog 内容，插件市场显示完整中英文双语版本
